# Backup and Recovery

## Backups
- Use `scripts/ops/backup-postgres.sh` to create logical backups.
- Use `scripts/ops/check-backup-health.ts` to fail fast if the latest backup is stale or if the WAL archive lag exceeds policy.

## Restore
- Use `scripts/ops/restore-postgres.sh --force <backup>` for explicit restore operations.
- Use `scripts/ops/refresh-staging.ts --env-file=<file> --backup-file=<dump> --apply` to restore staging and rehydrate it with the anonymized staging seed.

## WAL and PITR
- The F8 baseline assumes continuous WAL archival with a maximum lag of 5 minutes.
- Point-in-time recovery drills should capture the requested recovery point and the effective recovered point.
- Record the outcome with `scripts/ops/record-disaster-recovery-drill.ts`.

## RTO and RPO
- Target RPO: 5 minutes.
- RTO and achieved RPO must be measured per drill and stored as artifacts under `artifacts/backups/`.
- The repository does not invent production values; the measured values come from the drill artifact generated by `record-disaster-recovery-drill.ts`.

## Cadence
- Run a full disaster recovery drill every 6 months.
- Run the backup health check after each backup job and before each maintenance window.
